# /drug-interaction — 약물 상호작용 체크

약품명 입력 → 상호작용 분석 → 위험도 분류 → 복약 안내문 생성.

## WHEN TRIGGERED - EXECUTE IMMEDIATELY

- 약품명 인자 있으면 → 바로 Phase 0로
- 인자 없으면 → AskUserQuestion:
  "어떤 약품들의 상호작용을 확인할까요? 약품명을 쉼표로 구분해서 입력해주세요."
  (성분명 또는 상품명 모두 가능)

## Phase 0: 약품 식별 + 맥락 수집

입력된 약품명을 파싱하고, 각 약품에 대해:
- 성분명 (일반명)
- 대표 상품명
- 약효 분류
- 주요 적응증

AskUserQuestion:

**"[N개] 약품을 확인합니다. 어떤 상황인가요?"**
1. 처방전 검토 — 환자가 가져온 처방전 확인
2. 복합 복용 상담 — 환자가 현재 복용 중인 약 + 새 약 추가
3. OTC 상담 — 일반의약품/건기식 병용 확인
4. 단순 조회 — 상호작용만 빠르게 확인

## Phase 1: 상호작용 분석

약품 조합별 상호작용을 분석한다. WebSearch로 최신 정보를 확인.

### 상호작용 등급

| 등급 | 의미 | 조치 |
|---|---|---|
| ⚫ 금기 (Contraindicated) | 절대 병용 금지 | 즉시 처방의 확인 필요 |
| 🔴 심각 (Major) | 생명 위협 가능. 대안 검토 필요 | 처방의에게 연락 권고 |
| 🟡 중등도 (Moderate) | 모니터링 필요. 용량 조절 가능 | 복약 지도 시 주의사항 안내 |
| 🟢 경미 (Minor) | 임상적 영향 작음 | 참고 사항으로 안내 |
| ⬜ 상호작용 없음 | 알려진 상호작용 없음 | — |

### 분석 출력 형식

```
💊 약물 상호작용 분석
━━━━━━━━━━━━━━━━
약품 수: [N개]
상호작용 발견: [N건]

■ 약품 목록
| # | 성분명 | 상품명 | 약효 분류 |
|---|---|---|---|
| 1 | Amlodipine | 노바스크 | 칼슘채널차단제 |
| 2 | Metformin | 글루코파지 | 경구혈당강하제 |
| ... | ... | ... | ... |

■ 상호작용
| 약품 A | 약품 B | 등급 | 상호작용 내용 | 기전 |
|---|---|---|---|---|
| [약품1] | [약품2] | 🔴 심각 | [구체적 상호작용 내용] | [약리학적 기전] |
| [약품1] | [약품3] | 🟡 중등도 | [내용] | [기전] |

⚫ 금기: N건
🔴 심각: N건
🟡 중등도: N건
🟢 경미: N건
```

분석 직후 AskUserQuestion:
1. **⚫🔴 고위험 상호작용 상세 보기** (추천)
2. 대체 약품 제안
3. 복약 안내문 생성
4. 이 요약이면 충분

## Phase 2: 상세 분석 (선택 시)

```
━━━━━━━━━━━━━━━━
📌 [약품A] + [약품B] — 🔴 심각
━━━━━━━━━━━━━━━━

■ 상호작용
[구체적으로 무슨 일이 일어나는지]

■ 기전
[약리학적으로 왜 이런 상호작용이 발생하는지]

■ 임상적 영향
- 증상: [환자에게 나타날 수 있는 증상]
- 발현 시기: [복용 후 언제쯤 나타나는지]
- 심각도: [어느 정도 위험한지]

■ 대응
- 권고: [용량 조절 / 시간 간격 / 대체 약품 / 병용 금지]
- 모니터링: [어떤 증상/수치를 관찰해야 하는지]
- 대체 약품: [같은 약효의 상호작용 없는 대안]

■ 근거
- [약학 DB 또는 가이드라인 출처]
```

## Phase 3: 복약 안내문 생성 (선택 시)

환자에게 직접 줄 수 있는 안내문을 생성한다.

```
━━━━━━━━━━━━━━━━
📋 복약 안내문
━━━━━━━━━━━━━━━━

[환자 눈높이 언어로 작성]

■ 드시는 약
1. [약품명] — [용법: 하루 O회, 식후 O분]
2. [약품명] — [용법]

■ 주의사항
- [약품A]와 [약품B]는 [시간 간격]을 두고 드세요
- [구체적 주의사항을 쉬운 말로]
- 이런 증상이 나타나면 바로 약국/병원에 연락하세요: [증상]

■ 피해야 할 것
- [특정 음식/음료/활동]
- [이유를 쉬운 말로]
```

복약 안내문 생성 후 AskUserQuestion:
1. 파일로 저장 (마크다운)
2. 더 쉬운 말로 수정
3. 영어 버전도 생성
4. 완료

## Phase 4: 대체 약품 제안 (선택 시)

```
━━━━━━━━━━━━━━━━
💊 대체 약품 제안
━━━━━━━━━━━━━━━━

| 현재 약품 | 문제 | 대체 후보 | 동일 약효 | 상호작용 | 비고 |
|---|---|---|---|---|---|
| [약품A] | [약품B]와 금기 | [대체약1] | ✅ | ✅ 없음 | [참고사항] |
| | | [대체약2] | ✅ | 🟢 경미 | [참고사항] |
```

## 원칙

- 한국어 기본, 약품명은 성분명(일반명) + 상품명 병기
- **약사/의사의 전문적 판단을 대체하지 않는다** — "이 약을 드시면 안 됩니다" (X) → "이 조합은 심각한 상호작용이 보고되어 있어, 처방의 확인을 권고합니다" (O)
- 상호작용 등급은 반드시 기전(mechanism)과 함께 제시
- 복약 안내문은 **환자 눈높이** — 전문용어 없이, 구체적 행동 지시로
- 환자 정보(이름, 연락처, 질환)는 절대 기억/저장하지 않음
- 검토 결과 상단에 항상 표시: "⚠️ AI 보조 분석입니다. 최종 판단은 반드시 약사/의사와 확인하세요."
- 약품 정보는 WebSearch로 최신 정보를 확인하되, 출처를 명시
