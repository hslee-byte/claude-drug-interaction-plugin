# Contributing

Thanks for your interest in improving this project.

This repository is open source, but not every contribution will be merged. The maintainer keeps final control over scope, quality, and release direction.

## What This Project Optimizes For

- professional usefulness
- source traceability
- low-friction default UX
- concise outputs first, depth on demand
- safer wording in clinical contexts

## Preferred Contributions

- clearer evidence hierarchy
- better severity vs evidence-level separation
- lighter Lite-mode output
- better uncertainty handling
- more practical monitoring and counseling structure
- typo fixes and documentation improvements

## Contributions Likely To Be Rejected

- changes that make the default flow much heavier
- unnecessary clarification questions
- medication advice framed like a final clinical order
- unsupported alternative-drug recommendations
- changes that reduce source traceability

## Before You Open A PR

Please explain:

1. what problem you are solving
2. why this improves the workflow
3. whether Lite mode becomes heavier
4. how source quality or evidence clarity changes

## Review Principles

- Lite mode should stay fast and readable.
- Deep mode can be richer, but should stay structured.
- Severity and evidence level should remain separate.
- Uncertain clinical claims should stay explicitly uncertain.

## Small Practical Rule

If a proposal adds complexity, it should also reduce review time, error risk, or patient-facing confusion.
